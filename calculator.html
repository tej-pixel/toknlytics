How LLM cost is calculated

Every API call is priced per token — roughly ¾ of a word — and providers charge different rates for the tokens you send (input) and the tokens the model generates (output), usually quoted per million tokens. Your monthly bill is simply:

requests / month × (input tokens × input price + output tokens × output price)

Two things dominate the bill: which model you call and how many tokens each call uses. A frontier model can cost 20–50× a small one for the same task, and bloated prompts quietly inflate the input side. The calculator above shows the spread; the only way to know your real number — across every provider, with caching and batching factored in — is to measure it. That's what Meter does, free.

Cut the number you just saw

  • Right-size the model. Route easy calls to a smaller model; reserve the frontier model for the hard ones.
  • Trim prompts. Compress bloated system and context tokens — the input side is often the biggest, quietest cost.
  • Cache. Stop paying for repeat and near-duplicate calls.
