The privacy line

What we read — and what we never see

✓ We read

  • Token counts — input, output, cached
  • Model and endpoint names
  • Request timestamps & latency
  • Cost and billing lines
  • Error codes and retry signatures
  • Your team/project/customer tags

✗ We never see

  • Prompt content
  • Model outputs / completions
  • Your documents or embeddings content
  • End-user personal data
  • API keys with write scope — we can't change anything

The 7 providers

OpenAIAPI usage endpoints & billing exports
Connect an org-scoped, read-only API key. We poll usage endpoints — request counts, models, token totals — never your prompt or completion content.
AnthropicAdmin API usage & cost reports
Read-only admin key scoped to usage and cost reporting. Prompt caching stats included, so cache plays can be verified.
Google GeminiCloud Billing export & AI Studio usage
Read-only billing export (BigQuery or CSV) or AI Studio usage keys — whichever your setup already produces.
Llama (hosted)Gateway & host usage logs
For hosted Llama (Together, Groq, Fireworks and similar) we read the host's usage API; for self-hosted, a lightweight log shipper.
MistralLa Plateforme usage API
Read-only usage key. Same metadata-only rule: tokens, models, timestamps — no content.
Azure OpenAICost Management + deployment metrics
Read-only Cost Management scope plus deployment-level token metrics. Regional pricing differences feed the arbitrage play.
AWS BedrockCost Explorer + CloudWatch metrics
Cross-account read-only role for Cost Explorer and Bedrock CloudWatch metrics. No Lambda in your account, no write permissions.
